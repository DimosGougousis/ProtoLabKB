Agent Fleet Operations

Fleet-wide visibility into every agent's health, KPIs, cost, and compliance status. The real-time command center for your agent portfolio.

Why it matters

Without fleet-level visibility, each agent is a black box. Fleet operations gives you a single pane of glass across all agents, teams, and risk tiers. Health state machines detect degradation before users notice.

How we assess this

C1 (registry) + C2 (health every 30-60s) + C4 (monthly cost). Health state machine: healthy → degraded (error >5%) → critical (error >15% or circuit breaker open). T3 dashboard refreshes every 30s + WebSocket for state transitions.

Permission Boundaries

Three-layer tool sovereignty model: Configuration, Policy, and Network. Agents are first-class IAM citizens with Non-Human Identity, least-privilege scopes, and credential rotation.

Why it matters

An agent with unlimited access is a breach waiting to happen. Permission boundaries define exactly what each agent can do, what data it can access, and what tools it can call. Violations are caught by guardrails (C6) and decision logs (C7).

How we assess this

C1 defines boundaries (functions, outOfScope, permission_boundary_ref). C6 captures every enforcement event. C7 logs when agents exercise authority levels. NHI credentials tracked in nhi_credentials table with rotation policies.

Human Oversight Models

Three patterns scaled by risk: Human-in-the-Loop (HITL), Human-on-the-Loop (HOTL), Human-out-of-the-Loop Autonomy (HOTA). Each has different approval, latency, and cost characteristics.

Why it matters

EU AI Act mandates human oversight proportional to risk. HITL for every decision doesn't scale. HOTA without guardrails is reckless. The framework matches oversight model to risk tier and tracks actual human involvement via C7 decision logs.

How we assess this

C1 declares oversight model. C7 tracks each decision's authority level (A1-A5) showing actual human involvement. C3 traces show reasoning chains and where human checkpoints were hit or bypassed. Aggregated hourly for T2 CAIO dashboard.

Subsection 4 of 4

Delegation Chain Audit

When Agent A delegates to Agent B, which calls Tool C — who is accountable? Trace the full chain of authority, confidence, and monetary impact.

Why it matters

Multi-agent systems create accountability gaps. If a payment fails because Agent A delegated to Agent B who called an unreliable API, the delegation chain traces the full path. Every decision has an owner.

How we assess this

C7's delegatedTo field traces agent-to-agent delegations. C3 spans with parentSpanId reconstruct the call graph. Stored in delegation_chains + swarm_topology tables. Real-time via webhook on each delegation event.
